Overview

The core module of the shakespeare library provides a lightweight implementation of an actor model.
The main concepts present in this module are:

  • Behavior

    Interface defining an Actor behavior, that is, how the Actor reacts to start/stop events and newly received messages.
    The library is designed so that, every piece of code in a Behavior implementation is executed sequentially in such a way that no other synchronization mechanism is needed.
    A Behavior instance may represent a single state in a more complex state machine. In fact, it can be safely replaced at any moment during an Actor lifecycle.
    Inside a Behavior it is also possible to create new Actor instances, and dismiss them when needed.

  • Agent

    Interface defining an Actor execution environment.
    An Agent provides to the Behavior instances information about the Actor state and several ways to control it.
    Moreover, an Agent makes available references to utility classes such as a logger instance and executor services, employed to submit asynchronous tasks which are safely executed through the same actor synchronization mechanism.

  • Role

    Base class defining an Actor role.
    A Role provides the needed configuration to set up the newly created Actor instance and its initial Behavior.
